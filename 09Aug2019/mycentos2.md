```
파일 접근 권한

기호

chmod - 사용자 카테고리 문자 - 연산자 기호 - 접근 권한 문자 - 파일명

사용자 카테고리 문자 (u/g/o/a : 파일소유자/소유자 속한 그룹/ 기타 사용자/ 전체 사용자)

연산자 기호 (+/-/= : 권한부여/권한제거/접근 권한 설정)

접근권한 문자(r/w/k : 읽기/쓰기/실행)

ex) u+r => 소유자에게 읽기 권한 부여
    u-w => 소유자에게 쓰기 권한 삭제
    o-r => 기타 사용자(o)의 읽기(r) 권한 제거(-)
    u=rwx => 소유자에게 읽기/쓰기/실행 권한 부여(+)

숫자

소유자권한(읽기,쓰기,실행)-그룹권한(읽기,쓰기,실행)-기타사용자(읽기,쓰기,실행)

읽기는 2^2 = 4
쓰기는 2^1 = 2
실행은 2^0 = 1
으로 7이면 읽기쓰기실행의 권한을 가지는 것이다.

ex) 5 = 읽기 + 실행
    3 = 쓰기 + 실행
    6 = 읽기 + 쓰기
    4 = 읽기

chmod -R 777 = 소유자,그룹,기타사용자 모두 읽기,쓰기,실행 권한 가짐

umask(기본 접근 권한을 출력하거나 변경)

umask 옵션 마스크값

ex) 마스크값 022 / 일반파일 644 / 디렉토리 755 = 그룹과 기타사용자는 읽기만 가능

077 / 600 / 700 = 그룹과 기타사용자의 접근권한을 막음     


setUID : 해당 파일이 실행되는 동안에는 파일을 실행한 사용자의 권한이 아니라 파일 소유자의 권한으로 실행
파일에 SetUID 설정: SetUID는 접근 권한에서 맨 앞자리에 4를 설정

setGID : SetGID가 설정된 파일을 실행하면 해당 파일이 실행되는 동안에는 파일 소유 그룹의 권한으로 실행
SetGID는 2755와 같이 접근 권한에서 맨 앞자리에 2를 설정

스티키 비트

스티키 비트는 디렉터리에 설정
디렉터리에 스티키 비트가 설정되어 있으면 이 디렉터리에는 누구나 파일을 생성 가능
파일은 파일을 생성한 계정으로 소유자가 설정되며, 다른 사용자가 생성한 파일은 삭제 불가
/tmp 디렉터리가 대표적
스티키 비트는 접근 권한에서 맨 앞자리에 1을 설정
