# 머신러닝(테스팅 / 디버깅 3)

```
생산 시 파이프라인 테스트

훈련 점검 - 서비스 스큐

훈련 서비스 스큐는 입력 데이터가 다르다는 것을 의미 (훈련과 서빙 사이에 설명)

스키마 스큐 (교육 및 서비스 입력 데이터가 동인한 스키마를 준수하지 않음) -> 서비스 데이터의 형식 또는 분포가 변경 -> 동일한 스키마 사용 훈련 및 서비스 데이터 검증 

피쳐 스큐 (공학적 데이터는 교육및 서비스마다 다름 -> 기능 엔지니어링 코드는 훈련 및 서비스마다 다름 -> 스키마 스큐와 유사하게 동일한 통계 규칙을 전체 적용 / 엔지니어링은 데이터 제공 )

전체 파이프 라인에서 모델 사용기간 모델링 

서빙 데이터가 시간에 따라 변화 -> 모델이 정기적으로 재훈련 되지 않으면 모델 품질이 저하 / 서빙시 모델의 나리 감시 외에 노점을 잡기 위해 파이프라인 전체의 걸쳐 모델의 나이를 감시

모령 가중치와 출력이 출력이 숫자적으로 안정적인 지 테스트
모델 훈련 중에 체중과 레이어 출력은 NaN 또는 If가 아니어야함 -> 가중치와 레이어 출력의 NAN / inf 값을 테스트해야함! (레이어 출력의 절반 이상이 0이 아닌지 테스트하기)

모니터 모델 성능 


제공된 데이터에 대한 실시간 모델의 품질 테스트

모델 검증 / 검증 데이터 기록 -> 제공된 모델의 품질 저하 -> 실제 데이터가 항상 라벨로 표시되어 있는 게 아니기에 품질 테스트는 어려움 

예측에서 유의한 통계적 편향을 보이는 모형을 조사
모델에 대한 실제 메트릭을 추적
ex) 슾매 분류시 예측을 사용자가 보고한 스팸과 비교

문의 사항의 일부에 새 모델 버전을 제공하여 훈련 및 서비스 데이터 간의 잠재적 차이를 완화
새 서비스 모델을 검증하는 동안 모든 쿼리를 새 버전으로 점진적으로 전환
```
