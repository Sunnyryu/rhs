# 머신러닝(데이터 준비 / 피쳐 엔지니어링)

```
데이터준비

머신러닝은 데이터에서 패턴을 찾는데 도움 !(새로운 데이터 포인트에 대해 예측하는데 사용되는 패턴!)

예측을 제대로 하기 위해서는 데이터 세트 구성 / 올바르게 변환!

ex)신이 당신의 머신러닝 프로젝트에서 아래의 영역들 중 하나를 개선하는데 우선순위를 두어야 한다면, 어떤 것이 가장 큰 영향을 미칠 것?

데이터의 품질 및 크기
데이터가 모든 것을 능가한다. 학습 알고리즘이나 모델 아키텍처를 업데이트하면 다른 유형의 패턴을 학습할 수 있는 것은 사실이지만, 데이터가 나쁘면 결국 잘못된 것에 맞는 기능을 구축하게 된다. 데이터 세트의 품질과 크기는 당신이 사용하는 반짝이는 알고리즘보다 훨씬 더 중요

좋은 데이터 집합 수집이 중요한 이유

"관심 있어 보이는" 오류는 일반적으로 데이터에 의해 발생한다. 잘못된 데이터는 어떤 모델링 기법을 사용하든 상관없이 잘못된 패턴을 학습하게 할 수 있다.

process

ex) 머신러닝 프로젝트에서 데이터 준비 및 변환에 일반적으로 얼마나 많은 시간을 할애하시겠습니까?

프로젝트 시간의 절반 이상 / 데이터 세트 작성 / 데이터 변환

데이터 설계 단계

로우 데이터 수집 -> 라벨 소스/ 피쳐 식별 -> 샘플링 전략 선택 -> 데이터 분할

리뷰를 보지 않고 다운로드 했을지는 확실하지 않지만 다운로드 수와 리뷰 인상 횟수의 상관 관계가 있다고 하면? -> 리뷰를 보지 못한 사용자들의 행동을 비슷한 사용자들과 비교하는 실험 가능!

데이터 세트의 크기 / 품질

모델은 데이터 만큼 훌륭 -> 데이트 세트의 품질을 측정 / 개선

데이터 세트의 크기 

모델은 훈련 가능한 매개변수보다 최소한 더 많은 예제로 훈련!

데이터 세트의 상대적 크기를 고려할 필요가 있음!

나쁜 데이터라면 많은 데이터를 가져도 소용없음!
(경험적 접근 법을 취하고 최상의 결과를 산출하는 옵션을 선택!)

신뢰성, 특색 표현, 스큐 최소화

신뢰성은 데이터를 신뢰할 수 있는 정도 / 

데이터를 신뢰할 수 없는 이유 기계 학습 충돌 코스에서 다음 중 하나 이상으로 인해 데이터 세트의 많은 예제가 신뢰할 수 없다는 점을 상기

피쳐표현 

예측하지 못한 훈련을 하라. 즉, 당신의 훈련과제가 당신의 예측과 더 밀접하게 일치할수록 당신의 ML 시스템은 더 잘 수행

ML 목표는 고객 수를 특징으로 하여 일일 수익을 예측하는 것
```

```
데이터 로그

데이터 로그 가입
교육 세트를 구성할 때는 여러 데이터 출처에 가입해야 하는 경우

로그 유형

트랙잭션 로그, 데이터 속성, 집계하는 통계

트랜잭션로그는 특정 이벤트를 기록하며, 속성데이터에는 정보의 스냅샷이 포함되어 있음!

속성 데이터와 트랜잭션 로그는 관련이 있음

집계 통계는 여러 트랙잭션 로그에서 속성을 생성 (사용자 쿼리 빈도 ,특정 광고의 평균 클릭율 ,로그 소스 가입)

온라인—지속성이 문제가 되므로 시스템이 신속하게 입력을 생성해야 한다.
오프라인 -컴퓨팅에 제약이 없을 가능성이 높으므로 교육 데이터 생성과 마찬가지로 복잡한 작업을 수행할 수 있다.
```

```
레이블 및 소스 식별

직접 / 파생레이블

기계 학습은 라벨이 잘 정립되어 있을 때 더 쉽다.
가장 좋은 라벨은 당신이 예측하고 싶은 것의 직접적인 라벨

ex) 사용자가 테일러 스위프트 팬인지 여부를 예측하고 싶다면, 직접 라벨은 "사용자는 테일러 스위프트 팬"
팬덤 테스트는 사용자가 유투브에서 테일러 스위프트 영상을 봤는지 여부일 수 있음

레이블 소스
모델의 출력은 이벤트 또는 속성일 수 있음

"사용자가 상위 검색 결과를 클릭했는가?"와 같은 이벤트에 대한 직접 레이블
"광고주가 다음 주에 $X 이상을 지출할 것인가?"와 같은 속성 직접 레이블.
이벤트에 대한 직접 레이블

로그의 구조

사용자가 검색 결과를 클릭할 때 또는 사용자가 검색을 할 때 시스템이 로그에 기록

직접 레이블에 과거 동작 로그 필요

기록할 데이터가 없으면 어떻게 하시겠습니까?
아마도 당신의 제품이 아직 존재하지 않아서 기록할 데이터가 없을 것
데이터를 손으로 보는 것은 데이터를 어떻게 얻었든 상관없이 좋은 연습이다.
```
