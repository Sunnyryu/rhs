 하드디스크 관리와 사용자별 공간 할당

SATA / SCSI 장치

메인보드의 SATA 0 슬롯(메인보드에 케이블을 꽂을 수 있는 홈)

서버에서는 SATA보다는 SCSI를 마니 사용

RAID
여러 개의 하드디스크를 하나의 하드디스크처럼 사용하는 방식
  하드웨어 RAID와 소프트웨어 RAID로 나눔

하드웨어 RAID   
  하드웨어 제조업체에서 여러 개의 하드디스크를 연결한 장비를 만들어서 그 자체를 공급 (안정적임, 기술지원도 받을 수 있음, 고가)

소프트웨어 RAID
  하드웨어 RAID의 대안 / 하드디스크만 여러개 있으면 운영체제에서 지원하는 방식으로 RAID 구성 (신뢰성 속도가 떨어질 수 있으나 저렴한 가격)

RAID 레벨
  (Linear, 0,1,2,3,4,5 7가지로 구성됨)

단순볼륨 (하드디스크 하나를 볼륨 하나로 사용하는 방법 - RAID는 포함X)

LinearRAID / RAID 0
  최소 2개의 하드디스크 필요, 1개의 볼륨으로 사용
  그렇지만 Linear의 경우에는 파일이 저장되는 것이 앞 하드디스크에 데이터가 완전히 저장된 후에 다음 하드디스크에 데이터를 저장함

  0의 경우는 모든 하드디스크를 동시에 사용함.

  ex) LinearRAID : 안녕하세요? 써니예요 - 10초의 시간 발생
  ex) 0: 안하요써예/녕세니여? - 5초의 시간 발생

  0의 경우 속도가 빠르고 시간 효율이 좋음 그렇지만 1개가 고장나면 데이터를 못 읽음 !!, 리니어는 그거에 반대임!!(각 하드디스크 요량이 달라도 전체 요량을 문제 없이 사용할 수 있어 공간 효율성이 100%)

RAID 1
    미러링 방식임(똑같은 데이터의 거울을 만들어서 놓음. 하드디스크 2개에 했을 때 두개 다에 안녕하세요? 써니예요)
    데이터를 저장하는데 2배의 용량을 사용함/ 한개가 고장나도 손상이 안된다는 장점이 있음/ 공간 효율이 떨어짐

RAID 5
    최소 3개 이상의 하드디스크가 필요함/ 하드디스크가 오류가 발생해도 패리티를 통해서 데이터 복구 가능
    저장 공간의 효율이 좋음 / 하드디스크가 많을 수록 저장 공간의 효율을 더 올릴 수 있음/ 10개중에 2개가 고장나면 복구 못함
    패리티 1개 사용
RAID 6
    패리티를 2개 사용 / 공간효율은 5보다 떨어지지만 2개의 하드디스크가 고장이 나도 데이터에는 이상이 없음
    최소 4개의 하드디스크로 구성해야함, 신뢰도는 5에 비해 더 높음
    성능은 5보다 떨어짐 (쓰기 알고리즘이 복잡해지는 문제)

그 외에도 RAID 1로 구성한 데이터를 다시 RAID 0으로 구성하는 방법 등이 있음
