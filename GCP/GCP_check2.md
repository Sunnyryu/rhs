포드 : 특정 노드가 실행하는 개별 기능 단위가 동작하는 컨테이너 그룹, 포드를 구성하는 컨테이너는 모두 한 노드에 함께 유지되며, 동일한 IP주소와 포트 공간을 공유, 동일한 포드에 있는 컨테이너 들은 로컬호스트를 통해 통신가능 
그러나 동일한 포트에 바인딩을 할 수 없음 !
서비스는 다양한 포드가 실행되는 위치를 추적하는 데 사용하는 추상 개념 !

배포하기에 앞서 컨테이너가 구글 클라우드에 존재해야함 !

쿠버네티스 엔진에 배포하려면 쿠버네티스 클러스터를 설정해야함 !/ 쿠버네티스에는 기본 제공 Ui가 있으며, 쿠버네티스 엔진은 관리형 쿠버네티스 클러스터임..

Cloud Functions (서버리스 어플리케이션 )

마이크로 서비스 - 마이크로 서비스 아키텍처는 어플리케이션 조각을 느슨하게 결합하여 유지하는 어플리케이션을 빌드하고 조립! 

Functions를 사용하면 가상 서버, 컨테이너 또는 어플리케이션에 대하여 생각하는 대신 서버가 없는 환경에서 실행되는 단일 기능만 생각할 수 있음 ! , 서버에 어플리케이션 빌드, 배로, 필요한 디스크 공간을 생각하지 
않고 짧고 좁은 범위의 함수만 작성하며, 필요할 때마다 이러한 함수가 실행 !

HTTP 요청 외에도 요청을 트리거할 수 있음 !
파일 업로드나 Cloud Pub/Sub을 통해 게시되는 메세지에 의해 기능이 트리거 가능 !
Functions는 함수를 사용함 ... 함수를 다른 것에 연결하는 기능이 있음 !
이벤트는 어떤 것이 발생하면 발생(EX)Cloud Storage Object 생성), 함수( 이벤트에 응답하여 실행되는 코드 덩어리), 트리거(함수를 일부 이벤트에 연결하는 방법! ) 

이벤트 : http 요청 처럼 이벤트는 일어나는 일에 해당 / 함수가 실행될 수 있게... Storage나 PubSUB와 같이 다른 곳에 제공 될 수 있으며, 속성을 제공함 , Cloud Storage의 이벤트에는 영향을 받는 클라우드 스토리지 개체가 데이터 필드에 있음 
동기 이벤트와 비동기 이벤트로 나눔 

함수 : 한가지 기능 담당 .. 독립적으로 아무런 문제가 없어야함 , 클라우드 Function의 핵심 부분은 코드이다.. 이벤트 처리방식에 따라 함수의 구조가 약간 다를 수 있음 ... 비동기 이벤트를 처리하기 위해 작성된 함수는 백그라운드 함수라고 함 

트리거 : 접착제와 비슷 ... 라우트 되어야하는 이벤트를 지정.. 제공자를 기준으로 수행.. 특정 서비스의 이벤트 뿐만 아니라 이벤트를 원하는 리소스로 좁힐 수 있는 필터 지정 ..

마이크로서비스를 사용하면 별도의 독립 실행형 기능으로 어플리케이션을 구축가능.. gcp에 마이크로 서비스를 배포하고 실행하는 방법.. 함수 핸들러에는 동기함수와 비동기 두가지 유형에 있음 동기 함수는 http 요청에 응답 
함수는 트리거 등록.. pub/sub과 같은 다른 서비스에 이벤트를 전달 !!


Cloud DNS 

Google Cloud DNS는 DNS 서버 역할을 함!, BIND와 같은 다른 서버처럼 DNS 쿼리에 응답할 수 있는 관리형 서비스 ! / BIND 서버를 실행하지 않고 자신의 DNS 서버를 관리하기 위함 !!

레코드 갱신은 BIND 서비스를 실행 중인 시스템에서 파일을 수정한 다음 내용을 프로세스의 메모리로 다시 로드 !

DNS는 프로그램 할 수 있는 API를 제공하는 호스트 가능한 고가용성 DNS 서버 집합 !!
