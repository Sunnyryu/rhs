CREATE Procedure 이름(); 으로 만들고 CALL 이름(); 을 해서 호출해서 사용하면 되유..

ALTER Procedure가 수정이며, DROP Procedure를 하면 삭제를 할 수 있음 ..
IN 입력_매개 변수_이름 데이터_형식 으로 매개변수를 지정하며, CALL 프로시져_이름(전달값)을 사용하며, 출력 매개변수를 사용하기 위해서 => OUT 출력_매개변수_이름 데이터_형식을 사용한다. 대입하기 위해서는 SELECT ... INTO 문을 사용함 ..

특징으로는 DB의 성능을 향상시키고 모듈식 프로그래밍이 가능하며 유지관리가 수월하다는 것이다 !.

스토어드 함수는 => 사용자가 직접 만든 함수이다 .. / IN, OUT을 사용할 수 없으며, 모두 입력 파라미터이며, 리턴문(RETURNS)으로 반환할 값의 데이터 형식을 지정하고 본문 안에 RETURN 문으로 하나의 값을 반환해야함 !, 그리고 스토어드 함수는 SELECT 문장 안에서 호출한다..

AFTER 트리거는 DML 문의 작업이 일어날 때 작동하는 트리거이며 BEFORE 트리거는 이벤트가 발생하기 전에 작동하는 트리거이며, DML에서 일어남 ..

다중 트리거 . 하나의 테이블에 동일한 트리거가 여러 개 부착되어 있는 것을 말함 .. 한개 테이블에 2개 이상 부착되어 있을 수도 있음 ... 

중첩 트리거 .. 트리거가 또 다른 트리거를 작동하는 것을 말함 .. 예를 보면서 이해하면 낫베드 .. / FOLLOWS 트리거이름 => 지정한 트리거 다음에 현재 트리거가 작동되며.. PRECEDES 트리거이름 => 지정한 트리거가 작동하기 이전에 현재 트리거가 작동된다는 의미임 !
