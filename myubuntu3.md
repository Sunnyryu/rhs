네트워크 관련 설정과 명령어

네트워크 관련 필수 개념

TCP/IP
  프로토콜의 종류 중 하나
  통신의 전송/수신을 다루는 TCP(Transmission Control Protocol)와 데이터 통신을 다루는 IP(internet Protocol)로 구성됨.


호스트 이름과 도메인 이름
  호스트 이름은 각각의 컴퓨터에 지정된 이름이다.
  도메인 이름(도메인 주소)는 xxx.co.kr 같은 형식으로 표기하며 kr은 한국, co는 회사, xxx는 이름이다.
  호스트 이름을 포함한 주소는 FQDN이라고 함(Full Qualified Domain Name)이라고 부름

IP 주소
  각 컴퓨터의 랜 카드(네트워크 카드=NIC:Network Interface Card)에 부여되는 중복되지 않은 유일한 주소
  네트워크에 연결된 모든 컴퓨터는 고유한 ip주소가 있으며, 이는 서로 다르기 때문에 특정 컴퓨터의 IP 주소를 알면, 전 세계 어디에서든 접속 가능
  (사설 IP주소는 예외 )
  4바이트로 이루어져 있고 0~255까지 가능, server ip주소와 자기 자신을 의미하는 ip주소가 있음


네트워크 주소
  같은 네트워크에 속해 있는 공통 주소
  서버의 주소가 192.168.xxx.xxx, server(b)는 192.168.xxx.yyy, client가 192.168.xxx.zzzz이고 host 컴퓨터가 192.168.xxx.ttt이라고 할 때 공통된 네트워크 주소는 192.168.xxx 임

브로드캐스트 주소
  내부 네트워크의 모든 컴퓨터가 수신하느 주소 말함
  현재 주소의 제일 끝자리를 255로 바꾼 주소다 (c클래스의 경우)
  예로 필자의 브로드캐스트 주소는 192.168.xxx.255가 됨

게이트웨이
  게이트웨이는 내부 네트워크를 외부로 연결하기 위한 컴퓨터 또는 장비
  외부와 접속하기 위해 게이트웨이 ip 주소를 알아야만 함
  # route add default gw 게이트웨이주소 dev 장치이름

넷마스크와 클래스
  넷마스크는 네트워크의 규모를 결정
  사설 네트워크의 c 클래스를 사요하므로 넷마스크를 255.255.255.0으로 사용

DNS 서버 주소
  인터넷을 사용할 때 URL을 해당 컴퓨터의 IP주소로 변환해주는 서버 컴퓨터를 말함
  DNS 서버의 주소를 사용하지 않거나 잘못 입력되어 있으면 정상적으로 웹사이트에 접속되지 않으므로 올바른 정보를 설정해야함
  설정파일은 /etc/resolv.conf이며, nameserver DNS서버ip 형식으로 저장

중요한 네트워크 관련 명령어
  nm-connection-editor
    nm은 Network Manager의 약자로 네트워크 관련된 작업 대부분은 이명령을 기반으로 실행할 수 있음
    (자동 IP 주소 또는 고정 IP 주소 사용 결정, 정보 입력(IP주소 , 서브넷 마스크, 게이트웨이, DNS), 네트워크 카드 드라이버 설정 , 네트워크 장치 설정 )
  systemctl start/stop/restart/status network
    네트워크 설정을 한 후에 시스템에 적용하는 명령어
  ifconfig 장치이름
    해당 장치의 ip주소의 관련 정보 출력
  nslookup
    dns 서버의 작동 테스트 명령어
  ping ip 주소 또는 URL
    해당 컴퓨터가 네트워크 상에서 으답하는지를 테스트하는 간편한 명령어
네트워크 설정과 관련된 주요 파일
  /etc/resolv.conf
    DNS 서버의 정보와 호스트 이름이 들어있는 파일 (임시로 사용)
  /etc/hosts
    웹 컴퓨터의 호스트 이름과 FQDN이 들어있는 파일

파이프, 필터,리디렉션

파이프 : 2개의 프로그램을 연결시켜주는 연결 통로(|)
ex) ls -l /etc | less

필터 : 필요한 것만 길러주는 명령어다. grep, tail, wc, sort, awk, sed 명령어가 있음 (파이프와 사용)
ex) ps -ef | grep bash (ps -ef는 모든 프로세스 번호 출력 - 여기서는 bash라는 글자가 들어간 프로세서만 출력)

리디렉션 : 표준 입출력의 방향을 바꿔줌. 표준 입력은 키보드, 표준출력은 모니터 지만 이를 파일로 처리하고 싶을 때 해줌

ex) ls -l > list.txt (화면 출력 x , list.txt 파일에 저장하도록 함 기존에 있다면 덮여씀)

```
프로세스, 데몬, 서비스

프로세스 : 하드디스크에 저장된 실행 코드(프로그램)가 메모리에 로딩되어 활성화된 것
ex) firefox는 하드디스크의 어딘가에 저장되어 있을거임..
  포그라운드 프로세스
    방금 애기한 firefox 등과 같이 실행하면 화면에 나타나서 사용자와 상호작용을 하는 프로세스를 말함. 화면에서 실행되는 것이 보이는 프로세스!
  백그라운드 프로세스
    백그라운드 프로세스는 실해은 되었지만 화면에는 나타나지 않고 뒤에서 실행되는 프로세스 ( 백신, 서버 데몬 등 )
  프로세스 번호
    메모리에 로딩되어 활성화된 프로세스를 구분하려면 가각의 고유 번호가 필요함. 이렇게 각각의 프로세스에 할당된 고유 번호를 프로세스 번호라 부름 (메모리에서 활성화된 프로세스를 메모리에서 강제로 제거하려면 프로세스 번호를 사용해야함 )
  작업번호
    현재 실행되는 백그라운드 프로세스의 순차 번호
  부모 프레스와 자식 프로세스
    부모 프로세스의 하위에 종속되어 실행(모든 프로세스)
  명렁어
    ps (현재 프로세스의 상태를 확인하는 명령어)
      번호 및 상태를 확인 시 : ps -ef | grep 프로세스 이름
    kill
      프로세스를 강제로 종료하는 명령어 -9와 함꼐 사용
    pstree
      부모 프로세스와 자식 프로세스의 관계를 트리 형태로 보여줌  
 서비스와 소켓
  서비스 : 데몬이라고도 부름, 서버 프로세스임, 눈에 보이지 않지만 시스템에서 동작 중인 프로세스이므로 백그라운드 프로세스의 일종이라고 할수 있음(서비스 =데몬 = 서버프로세스?)

    특징 : 시스템과 독자적으로 구동되어 제공되는 프로세스
          실행 및 종료는 systemctl start/stop/restart 서비스 이름 명렁으로 사용
    명령어 :
            systemctl start/stop/restart 서비스 시작/중지/ 재시작
            systemctl status 서비스 이름 서비스 상태확인
            systemctl enable/disable 서비스 이름(서비스 사용/사용안함 설정 )
  소켓  
    외부에서 특정 서비스를 요청할 경우에 systemd가 구동시킴(요청 종료시 소켓 종료 )
    스크립트 파일은 /lib/systemd/system/ 디렉토리에 소켓이름.socket이라는 파일로 존재
